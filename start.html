Getting Started

Where to Download

The whole engine can be pulled from the github repository. Once pulled, the core engine is called "engine.js" and you will need to call it in your html before you call your game javascript.


External Dependencies

This engine does all collision and physics based calculations on its own, so it doesn't rely on any external libraries like Box2D.


File Organization

The "engine.js" file will can be kept wherever as long as you call it properly in your game's html. However, the texture folder needs to be in the same directory as the "engine.js" file. All of your sprites should be kept in your textures folder to be used properly.


How to Get Started

Our engine creates the stage, canvas, and event listeners on its own, so the only thing needed to be setup when beginning is an update function and a draw function. A call to the engine function "startGame(deltaT)" needs to be made at the end of the file to start the game. Update will always happen before draw in the main callback.
